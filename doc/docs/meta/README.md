# Documentation Meta

This folder is for documentation-system rules rather than product or
runtime knowledge.

Use it for:

- docs contribution rules
- naming and structure conventions
- pre-release verification

Current meta documents:

- [`./documentation-charter.md`](./documentation-charter.md) — the
  cross-tree charter defining audience, lifecycle, and boundaries for
  every documentation tree in the repository
- [`./contributing.md`](./contributing.md) — how to evolve the `docs/`
  tree
- [`./review-policy.md`](./review-policy.md) — what reviewers check on
  every pull request
- [`./release-gate.md`](./release-gate.md) — pre-release checklist
- [`./amendment-template.md`](./amendment-template.md) — canonical PR
  shape for amending the constitution or charter
