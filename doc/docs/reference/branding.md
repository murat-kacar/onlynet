# Branding

## Brand Stack

- **Company/Product:** `IoTable`
- **Category:** `Multi-tenant Admin Panel`
- **Platform UI:** `Management Console`
- **Operator Shell Pattern:** `Three-Pane Console Shell`

## Core Direction

- calm
- operational
- trustworthy
- clear

## One-Liner

`IoTable builds the operating system for modern table service.`

## Positioning

`IoTable` is a multi-tenant admin panel for running restaurant operations across customer environments.

## Product Line

| Surface | Name |
| --- | --- |
| platform admin | `IoTable Management Console` |
| tenant admin/staff | `IoTable Service Console` |
| guest ordering | `Tenant-branded menu` |

## Brand Split

- `IoTable` owns every operator-facing surface: platform, tenant admin, tenant staff, auth, and settings.
- Tenant branding appears only on the customer menu surface opened on the guest phone.
- Customer menu may use a warmer and more illustrative tone than the operator console surfaces.

## Voice

- short sentences
- low-hype language
- confident, not flashy
- practical over promotional

## Visual Direction

- light surfaces
- deep teal primary
- dense but orderly layouts
- no purple-gradient SaaS look
- scan-first, read-second
- state shown through hierarchy, color, and grouping
- fixed top bar, collapsible nav, and inspector-driven detail
- operator surfaces should read like a workbench, not a brochure

## UX Rules

- `★★★★★` operator rule: `inspector-first, overlay-for-actions`
- short labels, not paragraphs
- one dominant action per screen
- tables for control, cards for summary
- operators should scan, not study
- empty states stay useful and calm
- primary navigation lives in the left rail
- the current workspace title lives in a fixed top bar
- selection detail and secondary context live in a right inspector panel
- dashboards summarize system state; they do not duplicate sidebar navigation
- create, edit, and confirm actions prefer modal or drawer workflows over route jumps
- selected-record inspection prefers the right panel over route changes
- task-starting actions prefer overlays over hidden background state changes

## Reference Models

- `Shopify Plus Organization Admin` for multi-store / multi-tenant clarity
- `Stripe Dashboard` for calm operational density
- `Cloudflare` for control-plane seriousness
- `Okta Admin Console` for administrative trust and discipline
- `Figma` for inspector-panel mechanics
- `Notion` for collapsible navigation ergonomics
